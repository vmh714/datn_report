# Task Breakdown

Detailed instructions for decomposing screens into implementable components.

## When to use this skill

- Use this when starting a new feature or screen.
- This is helpful for creating a roadmap for FE and BE developers.

## How to use it

1. **Review Architecture**: Consult the **Solution Architect** or review their output from the `iot-system-architect-brainstormer` skill to understand the data flow and system components.
2. **Review instruction.md**: Align architectural decisions with the core project requirements and strict scope.
3. **Identify Components**: List UI components (FE) and API endpoints (BE).
4. **Define Workflows**: Mapping how firmware events trigger backend processing and frontend updates.
5. **Output Checklist**: Generate a detailed task list for the team.
